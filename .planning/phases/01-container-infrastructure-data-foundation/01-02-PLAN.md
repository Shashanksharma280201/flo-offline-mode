  <files>Dockerfile</files>

  <action>
Create Dockerfile in project root with two stages: builder (compile TypeScript) and production (run compiled app).

**Stage 1 - Builder:**
```dockerfile
# Stage 1: Builder
FROM node:22.13.1-alpine3.23 AS builder

WORKDIR /app

# Copy dependency manifests
COPY package.json pnpm-lock.yaml ./

# Enable pnpm and install all dependencies (including devDependencies for build)
RUN corepack enable && pnpm install --frozen-lockfile

# Copy source code
COPY tsconfig.json ./
COPY src ./src

# Compile TypeScript to JavaScript
RUN pnpm build
```

**Stage 2 - Production:**
```dockerfile
# Stage 2: Production
FROM node:22.13.1-alpine3.23

# Install tini for PID 1 signal handling (INFRA-03 requirement)
RUN apk add --no-cache tini

WORKDIR /app

# Copy dependency manifests
COPY package.json pnpm-lock.yaml ./

# Enable pnpm and install ONLY production dependencies
RUN corepack enable && pnpm install --frozen-lockfile --prod

# Copy compiled JavaScript from builder stage
COPY --from=builder /app/build ./build

# Run as non-root user for security
USER node

# Expose HTTP port
EXPOSE 3000

# Health check (placeholder - actual healthcheck.js created in Plan 04)
HEALTHCHECK --interval=30s --timeout=5s --start-period=40s --retries=3 \
  CMD node /app/build/healthcheck.js || exit 1

# Use tini as PID 1 to forward signals
ENTRYPOINT ["/sbin/tini", "--"]

# Start server
CMD ["node", "build/server.js"]
```

**Why these exact settings:**
- node:22.13.1-alpine3.23: Research confirms official LTS until April 2027, Alpine reduces to ~180MB
- tini: Research Pitfall 1 - required for SIGTERM forwarding to prevent data corruption
- --frozen-lockfile: Ensures reproducible builds (same versions in dev and prod)
- --prod: Excludes devDependencies in production stage (saves 50-70MB)
- USER node: Security best practice (non-root user)
- ENTRYPOINT tini: Pattern 3 from research - PID 1 signal handling
- HEALTHCHECK: Docker native health monitoring (Plan 04 creates actual healthcheck.js)

**Expected image size:**
~200-250MB total (node:22-alpine 180MB + prod deps 20-70MB). Meets <500MB requirement.
