  <action>
Create docker-compose.yml in project root with Docker Compose version 3.8 format.

**Services to create:**

1. **mongodb service:**
   - Image: mongo:8.0.20
   - Container name: flo-offline-mongodb
   - Restart policy: unless-stopped
   - Named volumes: mongodb_data → /data/db, mongodb_config → /data/configdb
   - Environment variables: MONGO_INITDB_ROOT_USERNAME=admin, MONGO_INITDB_ROOT_PASSWORD=${MONGO_PASSWORD}
   - Command: --config /etc/mongo/mongod.conf
   - Volume mount (read-only): ./docker/mongod.conf → /etc/mongo/mongod.conf:ro
   - Network: flo-offline-network
   - Health check:
     - test: ["CMD", "mongosh", "--eval", "db.adminCommand('ping')"]
     - interval: 10s
     - timeout: 5s
     - retries: 5
     - start_period: 40s
   - stop_grace_period: 60s (CRITICAL for MongoDB flush)

2. **redis service:**
   - Image: redis:7.2-alpine
   - Container name: flo-offline-redis
   - Restart policy: unless-stopped
   - Named volume: redis_data → /data
   - Command: redis-server --maxmemory 256mb --maxmemory-policy noeviction
   - Network: flo-offline-network
   - Health check:
     - test: ["CMD", "redis-cli", "ping"]
     - interval: 10s
     - timeout: 5s
     - retries: 3
     - start_period: 20s

3. **app service:**
   - Build context: . (current directory)
   - Dockerfile: ./Dockerfile (will be created in Plan 02)
   - Container name: flo-offline-app
   - Restart policy: unless-stopped
   - Ports: 3000:3000 (expose HTTP API)
   - Environment variables:
     - NODE_ENV=production
     - MONGO_URI=mongodb://admin:${MONGO_PASSWORD}@mongodb:27017/flo_offline?authSource=admin
     - REDIS_HOST=redis
     - REDIS_PORT=6379
   - Network: flo-offline-network
   - depends_on with conditions:
     - mongodb: condition: service_healthy
     - redis: condition: service_healthy
   - Health check:
     - test: ["CMD", "node", "/app/build/healthcheck.js"]
     - interval: 30s
     - timeout: 5s
     - retries: 3
     - start_period: 40s

**Named volumes to define:**
- mongodb_data (driver: local)
- mongodb_config (driver: local)
- redis_data (driver: local)

**Networks to define:**
- flo-offline-network (driver: bridge)

**Why these exact values:**
- MongoDB 8.0.20: Research shows 36% faster reads than 7.0, matches cloud performance
- Redis 7.2-alpine: BullMQ requirement, Alpine variant ~50MB
- stop_grace_period 60s: MongoDB needs time to flush writes to prevent corruption
- maxmemory-policy noeviction: BullMQ official requirement (research Pitfall 3)
- Health checks with start_period: Databases need 20-40s for first startup (research Pitfall 5)
- Named volumes NOT bind mounts: Production requirement for portability (research Pattern 2)

**Directory:** docker/

**File:** docker/mongod.conf

Content (YAML format):
```yaml
# MongoDB Configuration for Offline Mode Container
# This file will be fully configured in Plan 03 with WiredTiger cache settings

storage:
  dbPath: /data/db
  journal:
    enabled: true

net:
  bindIp: 0.0.0.0
  port: 27017

# WiredTiger cache configuration - to be set in Plan 03
# storage:
#   wiredTiger:
#     engineConfig:
#       cacheSizeGB: 0.5
```

**Why placeholder:**
Plan 03 will configure the actual WiredTiger cache size based on container memory limits (INFRA-04 requirement). This placeholder ensures Docker Compose can mount the config file without errors during initial testing.

**Why these defaults:**
- journal.enabled: true - Default in MongoDB 8.0, required for crash recovery
- bindIp: 0.0.0.0 - Container needs to accept connections from app service
- port: 27017 - MongoDB default, matches MONGO_URI in docker-compose.yml
